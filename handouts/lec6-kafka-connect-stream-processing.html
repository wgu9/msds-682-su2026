Choose the boundary

Start with the responsibility, not the tool

  • Integration moves records between systems.
  • Application I/O implements product behavior.
  • Computation derives facts and state.

Decision rule

Choose the smallest tool that owns the job

You need toStart with

04
Stream processing

Turn retained events into new facts

  • Kafka retains; the processor computes.
  • Own keys, state, and windows explicitly.
  • Acknowledge output before committing input.

Two processing shapes

State begins when one record is not enough

Stateless

One record in, one decision out

Validate, redact, convert units, enrich from fixed configuration.

Stateful

Output depends on prior records

Count, deduplicate, window, join, maintain a table.

State needs an owner: a key, a lifetime, and a recovery path.

Two interpretations

A stream remembers events; a table holds current state

ViewQuestionExample
StreamWhat happened, in what order?Requested, assigned, completed
TableLatest value for this key?Current trip status by trip_id
The same records support both views. The processor decides what to materialize.

Stateful ingredients

Key, state, and window make correlation explicit

KeyWhich entity owns the sequence?
+
StateWhat must survive between records?
+
WindowWhich interval belongs together?
Subscribing is not joining. Correlation needs keys, state, and time bounds.

Delivery boundary

Commit input only after output is acknowledged

ConsumeRead one input record
→
ValidateWire, then meaning
→
ProcessDerive the result
→
Output ackBroker confirms delivery
→
CommitStore next input offset
Commit means consumer offset commit. Not producer delivery. Not Git.

Failure reasoning

This ordering gives at-least-once, not exactly-once

The crash window

Output delivered, commit missed

After restart the same input is read again and may produce again.

The control

Stable event identity

A deterministic key makes repeated processing safe to deduplicate.

Do not promise exactly-once from commit order alone. Transactions have extra requirements.

Lecture 6 synthesis

Five ideas organize the whole system

  1. Connect moves records between systems.
  2. Kafka retains ordered records.
  3. A processor computes and may own state.
  4. Output acknowledgement precedes input commit.
  5. Resume continues; replay revisits history.
